Forums¶

Categories¶

POST /forums/categories¶

This is the endpoint for forum category creation. The forums_forums_modify permission is required to access this endpoint.

Example request:

POST /forums/categories HTTP/1.1

{
  "name": "Support",
  "description": "The place for confused share bears.",
  "position": 6
}

Example response:

{
  "status": "success",
  "response": "<ForumCategory>"
}
Response JSON Object:
 
  • response (dict) – The newly created forum category
Status Codes:
  • 200 OK – Creation successful
  • 400 Bad Request – Creation unsuccessful
GET /forums/categories¶

This endpoint allows users to view the available forum categories and the forums in each category, along with some information about each forum.

Example response:

{
  "status": "success",
  "response": [
    "<ForumCategory>",
    "<ForumCategory>"
  ]
}
Response JSON Object:
 
  • response (list) – A list of forum categories
Status Codes:
  • 200 OK – View successful
  • 401 Unauthorized – View unsuccessful
DELETE /forums/categories/(int: id)¶

This is the endpoint for forum category deletion . The forums_forums_modify permission is required to access this endpoint. The category must have no forums assigned to it in order to delete it.

Example response:

{
  "status": "success",
  "response": "ForumCategory 1 (Site) has been deleted."
}
Response JSON Object:
 
  • response (str) – The deleted forum category message
Status Codes:
  • 200 OK – Deletion successful
  • 400 Bad Request – Deletion unsuccessful
  • 404 Not Found – Forum category does not exist
PUT /forums/categories/(int: id)¶

This is the endpoint for forum category editing. The forums_forums_modify permission is required to access this endpoint. The name, description, and position of a forum category can be changed here.

Example request:

PUT /forums/categories/6 HTTP/1.1

{
  "name": "Support",
  "description": "The place for very confused share bears.",
  "position": 99
}

Example response:

{
  "status": "success",
  "response": "<ForumCategory>"
}
Response JSON Object:
 
  • response (dict) – The edited forum category
Status Codes:
  • 200 OK – Editing successful
  • 400 Bad Request – Editing unsuccessful
  • 404 Not Found – Forum category does not exist

Threads¶

This is the endpoint for forum thread creation. The forums_threads_modify permission is required to access this endpoint.

Example request:

POST /forums/threads HTTP/1.1
Host: pul.sar
Accept: application/json
Content-Type: application/json

{
  "topic": "How do I get easy ration?",
  "forum_id": 4
}

Example response:

HTTP/1.1 200 OK
Vary: Accept
Content-Type: application/json

{
  "status": "success",
  "response": {
  }

Polls¶

PUT /polls/(int: id)¶

This is the endpoint for forum poll editing. The modify_forum_polls permission is required to access this endpoint. The choices can be edited here, as well as the closed and featured statuses.

Example request:

PUT /forums/polls/6 HTTP/1.1

{
  "id": 1,
  "choices": {
    "add": ["Yes", "No"],
    "delete": ["Placeholder1", "Placeholder2"]
  },
  "closed": true,
  "featured": false
}

Example response:

{
  "status": "success",
  "response": "<ForumPoll>"
}
Response JSON Object:
 
  • response (dict) – The edited forum poll
Status Codes:
  • 200 OK – Editing successful
  • 400 Bad Request – Editing unsuccessful
  • 404 Not Found – Forum poll does not exist
